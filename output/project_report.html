5.2 ESP32 to Flask Communication

What ESP32 Sends (Arduino Code)

// ESP32 Arduino — runs every ~20 seconds
#include <WiFi.h>
#include <HTTPClient.h>

float distance = measureUltrasonic();  // HC-SR04 reading

HTTPClient http;
http.begin("http://<RPi-IP>:5000/sensor");
http.addHeader("Content-Type", "application/json");

String payload = "{\"distance_mm\": " + String(distance) + 
                 ", \"device_id\": \"ESP32-Sensor-01\"}";
int status = http.POST(payload);
String response = http.getString();  // Detection result JSON
http.end();

What Flask Receives & Does

# Flask endpoint: POST /sensor
# 1. Receives: {"distance_mm": 42.5, "device_id": "ESP32-Sensor-01"}
# 2. Appends to rolling buffer (last 200 readings)
# 3. Runs MA+SD detection -> flag
# 4. Runs Rate of Change -> flag
# 5. Runs Adaptive CUSUM -> flag
# 6. Computes confidence score (0.0 - 1.0)
# 7. Classifies anomaly type
# 8. Stores in SQLite (readings table)
# 9. If score >= 0.4 -> stores in alerts table + sends notification
# 10. Returns JSON result to ESP32

What Flask Returns

{
  "distance_mm": 42.5,
  "level_pct": 83.0,
  "masd_flag": true,
  "roc_flag": false,
  "cusum_flag": true,
  "n_methods": 2,
  "conf_score": 0.40,
  "conf_tier": "alert",
  "anomaly_class": "sensor_spike",
  "flow_rate": 0.15,
  "flow_direction": "stable"
}

12.3 How to Connect ESP32 for Real-Time (Quick Start)

Steps to go from CSV mode to real-time:
  1. Connect HC-SR04 to ESP32 (Trigger=GPIO 5, Echo=GPIO 18)
  2. Flash Arduino code that POSTs to http://<RPi-IP>:5000/sensor
  3. Start Flask on Raspberry Pi: python 06_flask_backend.py
  4. Open dashboard in browser: http://<RPi-IP>:5000/dashboard
  5. Data flows automatically — readings appear in real-time!
