DEPLOYMENT ANALYSIS -- TinyML on ESP32
=================================================================

Target hardware: ESP32 DevKit v1 (240 MHz, 520 KB SRAM)
  Available RAM: ~280 KB (after WiFi/BT stack)

Detector              RAM (bytes)  Inference (us)  Model (bytes)
-------------------- ------------ --------------- --------------
MA+SD                         416             ~15    0 (formula)
Rate of Change                  4              ~5    0 (formula)
Adaptive CUSUM                 12             ~10    0 (formula)
Prophet (lookup)              104              ~2     96 (array)
-------------------- ------------ --------------- --------------
TOTAL                         536             ~32             96

Memory usage: 536 bytes = 0.52 KB
  Percentage of ESP32 SRAM: 0.191%
  Percentage of ESP32 flash: negligible

Metric                                 ESP32 (C)  RPi Zero (Py)   RPi 4 (Py)
----------------------------------- ------------ -------------- ------------
Statistical inference (us)                   ~32           ~500         ~200
STL + Prophet (ms)                           N/A           ~500         ~250
Throughput (readings/sec)                 ~30000          ~2000        ~4000
RAM for detection                         ~350 B         ~50 MB       ~50 MB
Power consumption                        ~315 mW         ~1.5 W         ~3 W
Unit cost (INR)                           Rs.500        Rs.1500      Rs.3500
WiFi required?                               No*            Yes          Yes

* ESP32 TinyML can detect anomalies locally without network connectivity.
  Results are buffered and sent when WiFi is available.

Conclusion:
  Statistical detectors (MA+SD, RoC, CUSUM) fit within 536 bytes
  of RAM on an ESP32 -- 0.13% of available memory. This leaves >99%
  of resources for WiFi stack, sensor drivers, and application logic.
  The detectors achieve ~32us inference time, enabling real-time
  anomaly detection at the sensor edge with no cloud dependency.
